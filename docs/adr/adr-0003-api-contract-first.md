# ADR-0003 API 계약 우선(OpenAPI SSOT) 채택

- 상태: accepted
- 작성일: 2026-02-16
- 의사결정자: Platform Team

## 배경/문제
프론트/백엔드가 독립적으로 DTO를 관리하면 타입 불일치로 런타임 오류와 재작업이 반복된다.

## 선택지
1. 코드 우선(구현 후 문서화)
2. 수동 API 문서 + 수동 타입 정의
3. OpenAPI SSOT + 코드 생성

## 결정
OpenAPI를 단일 진실 공급원으로 사용하고, 백엔드는 oapi-codegen, 프론트는 Orval 생성 코드를 사용한다.

## 근거
- 계약 불일치를 컴파일 단계에서 조기 검출
- 보일러플레이트 감소
- API 변경 영향 추적 용이

## 결과/영향
- 장점: 타입 안전성 향상, 협업 비용 감소
- 단점: 생성 파이프라인 유지 필요
- 후속 작업: CI에 계약 동기화 검증 단계 추가
