# ADR-0002 백엔드 헥사고날 아키텍처 채택

- 상태: accepted
- 작성일: 2026-02-16
- 의사결정자: Backend Team

## 배경/문제
Gin, DB, 외부 서비스 연동 코드가 유스케이스와 혼합되면 테스트가 어렵고 인프라 변경 시 대규모 수정이 발생한다.

## 선택지
1. 레이어드 MVC 구조
2. 패키지 by feature + 느슨한 인터페이스
3. 헥사고날(Ports & Adapters)

## 결정
도메인/애플리케이션/어댑터를 분리한 헥사고날 구조를 채택한다.

## 근거
- 도메인 순수성 보장
- 인프라 교체 비용 축소
- 유스케이스 중심 테스트 작성 용이

## 결과/영향
- 장점: 테스트성/유지보수성 향상, 의존성 방향 명확
- 단점: 초기 보일러플레이트 증가
- 후속 작업: 포트 인터페이스 정리, DI 조립 모듈 표준화
