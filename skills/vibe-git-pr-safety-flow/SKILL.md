# Git/PR 안전 흐름

## 목표
작업 흔적을 명확히 남기고 리뷰 가능한 단위로 변경을 제어한다.

## 실행 절차
1. 작업 목적에 맞는 브랜치를 만든다.
2. 변경을 의미 단위 커밋으로 분리한다.
3. 커밋 전 테스트/린트 결과를 첨부한다.
4. PR은 우선 Draft로 열고 체크리스트를 채운다.
5. 리뷰 코멘트 반영 시 재검증 후 force push 여부를 명시한다.

## PR 본문 최소 구조
- 배경
- 변경 내용
- 검증 방법(명령 + 결과)
- 리스크/롤백
- 미해결 이슈

## 가드레일
- 검증 로그 없이 "테스트 완료"라고만 쓰지 않는다.
- unrelated change를 같은 PR에 섞지 않는다.
- 급한 핫픽스도 최소 재현/검증 증거를 남긴다.
