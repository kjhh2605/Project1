# 테스트 우선 구현 루프

## 목표
빠른 구현보다 검증 가능한 구현을 우선해 장기 유지보수 비용을 낮춘다.

## 루프
1. 실패하는 테스트를 먼저 작성한다.
2. 최소 변경으로 테스트를 통과시킨다.
3. 관련 테스트 전체를 실행한다.
4. 중복만 제거하는 수준으로 정리한다.
5. 변경 범위를 재검토하고 커밋 단위를 분리한다.

## 필수 체크
- 재현 테스트 없이 버그를 수정하지 않는다.
- 테스트가 지나치게 구현 상세에 결합되지 않게 작성한다.
- 테스트 통과 후 성능/보안/동시성 영향이 있으면 추가 검증한다.

## 권장 명령 패턴
- 단위: 프로젝트별 test 명령
- 정적 검사: lint/typecheck
- 통합: 최소 핵심 시나리오

## 실패 대응
- 테스트 설계가 모호하면 요구사항 명확화 스킬을 먼저 적용한다.
- flaky 테스트는 원인 분석 없이 재시도 반복으로 덮지 않는다.
