# 풀스택 품질 게이트

## 목표
"돌아가는 것처럼 보임" 상태를 제거하고 릴리즈 신뢰도를 높인다.

## 필수 게이트(머지 전)
1. 정적 검사: lint + typecheck
2. 단위 테스트: 프론트/백엔드 핵심 로직
3. 계약 검증: OpenAPI 생성물 동기 상태
4. 통합/E2E: 핵심 사용자 흐름 1~3개
5. 빌드 검증: 배포 산출물 생성 가능 여부

## 테스트 전략
- Go: 유스케이스 단위 테스트 + 핸들러 `httptest`
- Frontend: 슬라이스 단위 테스트 + 사용자 흐름 E2E
- E2E는 핵심 시나리오에서 실제 백엔드와 연결해 검증한다.

## 보안/환경 규칙
- 프론트 공개 변수는 `NEXT_PUBLIC_`만 클라이언트 노출 허용.
- 비밀값은 `.env.local`에 두고 버전관리 금지.
- 템플릿 파일(`.env.example`)로 필수 키를 명시한다.
