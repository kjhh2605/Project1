# 성공 기준 기반 계획

## 목표
명령형 요청을 측정 가능한 목표로 바꿔 자동 반복(수정→검증)을 가능하게 한다.

## 실행 절차
1. 요청을 `완료 조건`으로 변환한다.
2. 완료 조건을 `관찰 가능한 체크`로 쪼갠다.
3. 작업을 3~7단계 계획으로 작성한다.
4. 각 단계에 검증 명령을 붙인다.
5. 실패 시 롤백 또는 대안 경로를 명시한다.

## 출력 템플릿
1. [작업 단계] → verify: [명령/체크]
2. [작업 단계] → verify: [명령/체크]
3. [작업 단계] → verify: [명령/체크]

완료 기준:
- [ ] 기준 1
- [ ] 기준 2
- [ ] 기준 3

## 예시 변환 규칙
- "버그 고쳐" → "재현 테스트 작성 후 통과"
- "유효성 검사 넣어" → "잘못된 입력 테스트 3종 추가 후 통과"
- "리팩터링" → "리팩터링 전/후 동등 동작 검증"

## 가드레일
- "잘 동작함" 같은 주관식 완료 기준을 금지한다.
- 검증 불가능한 목표는 수락하지 않고 재정의한다.
