# 외과적 변경 가드

## 목표
무관한 수정, 스타일 드리프트, 과도한 리팩터링으로 생기는 기술 부채를 방지한다.

## 실행 절차
1. 수정 대상 파일/라인의 근거를 요청과 연결한다.
2. 요청과 무관한 코드 정리는 분리 제안만 하고 즉시 수정하지 않는다.
3. 내 변경으로 생긴 orphan(미사용 import/변수)만 정리한다.
4. 기존 코드 스타일을 따르고 포맷 대변환을 피한다.
5. diff를 요청 기준으로 자체 감사한다.

## 자체 감사 체크리스트
- [ ] 각 변경 라인이 요청과 직접 연결되는가?
- [ ] 기능 외 변경(네이밍/포맷/구조)이 섞였는가?
- [ ] 영향 범위를 설명할 수 있는가?
