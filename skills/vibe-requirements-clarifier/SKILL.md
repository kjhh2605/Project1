# 요구사항 명확화

## 목표
요구사항 오해로 인한 재작업과 기술 부채를 줄인다.

## 실행 절차
1. 사용자 요청을 한 문장으로 재서술한다.
2. 불확실한 지점을 `가정`으로 명시한다.
3. 가능한 해석이 2개 이상이면 옵션 A/B로 제시한다.
4. 각 옵션의 비용(개발량), 리스크(회귀 가능성), 효과(문제 해결력)를 비교한다.
5. 구현 전 확정 질문을 1~3개로 압축해 제시한다.

## 출력 형식
다음 구조를 고정 사용한다.

- 요청 요약:
- 현재 가정:
- 대안:
  - A: 
  - B: 
- 권장안:
- 확인 필요 질문:

## 가드레일
- 확신 없는 부분을 추측으로 확정하지 않는다.
- "일단 만들고 보자" 방식으로 진행하지 않는다.
- 질문은 최소화하되, 아키텍처/데이터 모델/외부 인터페이스에 영향 주는 항목은 반드시 확인한다.
