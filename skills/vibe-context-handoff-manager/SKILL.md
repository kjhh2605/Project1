# 컨텍스트 핸드오프 관리

## 목표
긴 대화에서 품질 저하를 줄이고, 다음 세션/다음 에이전트가 즉시 이어받게 한다.

## 사용 기준
- 맥락 사용량이 높아져 응답 품질이 떨어질 때
- 작업 주제가 바뀔 때
- 병렬 작업 탭/세션을 운영할 때

## HANDOFF.md 최소 항목
- 목표
- 현재 상태(완료/진행/미시작)
- 시도한 방법과 결과
- 실패 원인/피해야 할 접근
- 다음 단계(우선순위 포함)
- 검증 명령

## 운영 규칙
1. 큰 단계 종료 시 HANDOFF.md를 업데이트한다.
2. 새 세션은 HANDOFF.md만 읽고 시작 가능해야 한다.
3. 미확정 가정은 별도 섹션으로 분리한다.
4. 완료 후 문서를 최신 상태로 정리한다.
