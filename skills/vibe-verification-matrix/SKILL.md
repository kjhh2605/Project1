# 검증 매트릭스

## 목표
검증 누락으로 생기는 잠복 결함을 줄이고, 배포 품질을 안정화한다.

## 매트릭스 축
- 범위: 단위 / 통합 / E2E / 수동 UX
- 관점: 기능성 / 성능 / 보안 / 회귀
- 환경: 로컬 / CI / 스테이징

## 실행 절차
1. 변경 영향도를 기반으로 필요한 셀만 선택한다.
2. 각 셀에 검증 방법과 통과 기준을 적는다.
3. 실행 결과를 pass/fail + 로그 링크로 남긴다.
4. fail 셀은 수정 후 동일 셀 재검증한다.

## 최소 산출물
| 항목 | 방법 | 통과 기준 | 결과 |
|---|---|---|---|
| 단위 테스트 | 명령 | 전부 통과 |  |
| 타입/린트 | 명령 | 에러 0 |  |
| 핵심 시나리오 | 수동/자동 | 기대 결과 일치 |  |

## 가드레일
- 증거 없는 통과 선언 금지.
- CI 통과만으로 실제 사용자 플로우 검증을 대체하지 않는다.
