# 기술 부채 방화벽 설계

## 목표
초기 속도를 유지하면서도 구조적 부채 누적을 통제한다.

## 1) 경계 정의
- 도메인 경계(프론트/백엔드/공유 모델)를 문서화한다.
- 외부 의존성 도입 기준(필요성, 대안, 유지비)을 정의한다.
- API/이벤트 계약을 단일 소스로 관리한다.

## 2) 품질 게이트
- 머지 전 필수 게이트: test, lint, typecheck, build.
- 실패 허용 기준을 예외 정책으로 분리한다.
- 임시 우회는 만료일과 제거 조건을 기록한다.

## 3) 부채 레지스트리
- TODO를 흩뿌리지 말고 `docs/tech-debt.md`에 집중 기록한다.
- 항목별로 원인/영향/해결비용/우선순위를 관리한다.
- 스프린트마다 상위 부채를 상환한다.

## 4) 리팩터링 트리거
다음 조건 중 하나면 리팩터링 티켓을 즉시 생성한다.
- 동일 코드 복붙 3회 이상
- 수정 1건에 파일 다수 동시 수정 필요
- 버그 수정 시 연쇄 회귀 발생

## 5) 의사결정 기록
중요 결정은 ADR 형태로 축약 기록한다.
- 문제
- 선택지
- 결정
- 근거
- 후속 영향
